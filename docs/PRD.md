# 四川农业大学校园闲置物品交易系统 产品需求文档（PRD）

**作者:** Ajie  
**日期:** 2025-10-27  
**项目级别:** Level 3 (Brownfield)  
**目标规模:** 12-40 个用户故事，2-5 个 Epic  
**基础架构:** litemall (Spring Boot + Vue.js + 微信小程序)

---

## 1. 目标与背景

### 1.1 产品目标

构建四川农业大学专属的校园闲置物品交易平台，通过微信小程序提供高效、安全、具有校园特色的二手交易服务，解决传统平台在垂直化服务、用户信任和物流配送上的痛点。

**核心目标:**
- **提升交易效率**: 将校园内二手交易周期从平均 7 天缩短至 3 天以内
- **建立信任机制**: 通过学号实名认证和信用积分体系，交易纠纷率控制在 2% 以下
- **创造校园价值**: 学生快递员配送覆盖三校区，公益捐赠通道促进资源再分配

### 1.2 背景与市场分析

**市场规模:**
- 2022 年中国二手电商交易规模达 4802.04 亿元，同比增长 20%
- 校园二手电子产品交易占比 40%，二手图书交易量达 1 亿册
- 微信小程序日活跃用户突破 4.5 亿，日均使用次数同比增长 32%

**用户痛点:**
- **信息不对称**: 线下交易缺乏透明度，存在质量欺诈风险
- **交易效率低**: 大众平台缺乏校园场景适配（如教材快速匹配、宿舍楼栋配送）
- **信任成本高**: 综合平台纠纷率较高，缺乏校园专属信用体系
- **物流不便**: 第三方快递无法覆盖宿舍内部，自提时间协调困难

**政策支持:**
- 国家"十四五"循环经济发展规划鼓励"互联网+二手"模式
- 2022 年二手手机交易实现碳减排 500 万吨，助力"双碳"目标

### 1.3 竞品分析

| 平台 | 优势 | 劣势 | 差异化策略 |
|-----|------|------|-----------|
| **闲鱼** | 用户规模大（5亿+）| 缺乏校园垂直化服务 | 强制学号认证限定校内交易 |
| **转转** | 质检服务完善 | C2B2C 模式存在压价问题 | C2C 直接交易，学生快递员配送 |
| **校园BBS** | 社区氛围浓厚 | 功能简陋，无支付保障 | 集成微信支付，信用互评体系 |

## 2. 用户画像与使用场景

### 2.1 核心用户画像

#### 用户角色 1: 大一新生（高频买家）
- **人口统计**: 18-19 岁，经济能力有限
- **核心需求**: 低价购买教材、生活用品、电子产品
- **痛点**: 担心被骗，不熟悉校园环境，物流不便
- **使用场景**: 开学季批量购买教材、入冬购买取暖设备

#### 用户角色 2: 毕业生（高频卖家）
- **人口统计**: 22-23 岁，即将离校
- **核心需求**: 快速清理闲置物品，回收部分成本
- **痛点**: 时间紧迫，传统平台成交周期长
- **使用场景**: 毕业季集中处理家电、书籍、衣物

#### 用户角色 3: 在校生（买卖双向）
- **人口统计**: 19-22 岁，有一定经济来源
- **核心需求**: 购买专业书籍、季节性商品（如考研资料、运动器材）
- **痛点**: 需要精准匹配（如指定课程的教材）
- **使用场景**: 期末购买复习资料、换季处理衣物

#### 用户角色 4: 勤工俭学学生（快递员）
- **人口统计**: 19-22 岁，希望通过兼职补贴生活费
- **核心需求**: 灵活接单，收入透明
- **痛点**: 传统兼职时间固定，收入不确定
- **使用场景**: 课余时间接单配送，每周赚取 50-200 元

### 2.2 典型用户旅程

#### 旅程 1: 新生购买二手教材
```
需求产生 → 搜索课程名（如"高等数学A"）→ 系统匹配教材列表 
→ 查看商品详情（价格/新旧程度/卖家信用）→ 联系卖家确认 
→ 下单支付 → 选择配送方式（自提/学生快递员）→ 确认收货 
→ 双方互评 → 完成交易
```

**关键触点优化:**
- **搜索**: 支持课程名直接匹配教材，无需输入 ISBN
- **信任建立**: 显示卖家信用等级和历史评价
- **配送选择**: 默认推荐同楼栋学生快递员

#### 旅程 2: 毕业生快速清仓
```
拍照上传多件商品 → 批量设置分类/价格 → 选择"限时拍卖"模式 
→ 系统推送至首页推荐位 → 买家竞拍 → 拍卖结束自动生成订单 
→ 学生快递员上门取件 → 买家确认收货 → 资金结算
```

**关键功能:**
- **批量发布**: 支持一次上传 5 件以上商品
- **拍卖机制**: 激发用户活跃度，提升成交价
- **上门取件**: 减轻毕业生物流负担

#### 模块 3: 交易流程管理
**复用 litemall 模块**: `litemall-order`（订单表、订单商品表）

| 功能点 | 需求描述 | 优先级 |
|-------|---------|--------|
| **下单支付** | 调用微信支付 JSAPI，支持优惠券抵扣（复用 litemall 优惠券表）| P0 |
| **配送方式** | 自提（买卖双方线下约定）/ 学生快递员配送（额外支付配送费）| P0 |
| **订单状态** | 待发货（24h 未发货提醒）→ 待收货（7 天自动确认）→ 待评价（15 天关闭）→ 已完成 | P0 |
| **取消订单** | 买家付款后 5 分钟内可无责取消，超时扣 5 积分；卖家接单后不可取消，违约扣 50 积分 | P0 |
| **互评系统** | 双方评分（1-5 星）、标签（描述相符/配送及时/态度友好）、文字评价（≤200 字）| P0 |

**订单状态流转图:**
```
创建订单 → 买家支付 → 待发货（卖家确认/快递员接单）
                   ↓
             待收货（买家确认收货/7 天自动）
                   ↓
             待评价（双方互评/15 天关闭）
                   ↓
              已完成
```

**技术实现:**
- 定时任务扫描超时订单（Spring `@Scheduled`）
- 微信支付回调验证（签名校验防篡改）

### 3.2 创新功能（校园特色）

#### 模块 4: 学生快递员配送系统

**业务规则:**
| 规则 | 详细说明 |
|-----|---------|
| **快递员资格** | 信用等级 ≥ ⭐⭐、无违规记录、提交学生证照片 |
| **配送范围** | 校内（成都校区、雅安校区、都江堰校区），按楼栋划分配送区域 |
| **接单限制** | 同一时段最多接 5 单，避免超负荷 |
| **配送费率** | 校内同校区 2 元/单、跨校区 5 元/单（买卖双方各承担 50%）|
| **时效要求** | 校内 2 小时、跨校区 24 小时 |
| **惩罚机制** | 超时 1 次警告，3 次取消资格；连续 10 单准时奖励 10 积分 |

**新增数据表:**
```sql
CREATE TABLE `sicau_courier` (
  `id` INT PRIMARY KEY AUTO_INCREMENT,
  `user_id` INT NOT NULL COMMENT '关联用户ID',
  `dorm_building` VARCHAR(50) COMMENT '宿舍楼栋（如"7舍A栋"）',
  `campus` ENUM('成都','雅安','都江堰') COMMENT '所在校区',
  `status` TINYINT DEFAULT 1 COMMENT '0申请中/1可接单/2已禁用',
  `total_orders` INT DEFAULT 0 COMMENT '累计配送单数',
  `timeout_count` INT DEFAULT 0 COMMENT '超时次数',
  `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE `sicau_courier_order` (
  `id` INT PRIMARY KEY AUTO_INCREMENT,
  `order_id` INT NOT NULL COMMENT '关联订单ID',
  `courier_id` INT COMMENT '快递员ID',
  `pickup_building` VARCHAR(50) COMMENT '取件楼栋',
  `delivery_building` VARCHAR(50) COMMENT '送达楼栋',
  `fee` DECIMAL(10,2) COMMENT '配送费',
  `status` TINYINT COMMENT '0待接单/1配送中/2已送达',
  `accept_time` DATETIME COMMENT '接单时间',
  `deliver_time` DATETIME COMMENT '送达时间'
);
```

#### 模块 5: 限时拍卖模式

**业务规则:**
| 规则 | 详细说明 |
|-----|---------|
| **发起条件** | 卖家信用等级 ≥ ⭐⭐，设置起拍价、加价幅度、拍卖时长（1-7 天）|
| **保证金** | 卖家缴纳起拍价 10% 作为保证金（拍卖结束退回）|
| **竞拍规则** | 买家出价需缴纳保证金（= 当前出价），被超越者保证金原路退回 |
| **延时规则** | 拍卖结束前 5 分钟内有新出价，自动延长 5 分钟 |
| **结算** | 最高出价者 24 小时内付款，否则扣除保证金并取消资格，第二高出价者递补 |

**新增数据表:**
```sql
CREATE TABLE `sicau_auction` (
  `id` INT PRIMARY KEY AUTO_INCREMENT,
  `goods_id` INT NOT NULL COMMENT '商品ID',
  `start_price` DECIMAL(10,2) COMMENT '起拍价',
  `current_price` DECIMAL(10,2) COMMENT '当前价格',
  `increment` DECIMAL(10,2) COMMENT '加价幅度',
  `end_time` DATETIME COMMENT '结束时间',
  `winner_id` INT COMMENT '中标用户ID',
  `status` TINYINT COMMENT '0进行中/1已结束/2已成交'
);

CREATE TABLE `sicau_auction_bid` (
  `id` INT PRIMARY KEY AUTO_INCREMENT,
  `auction_id` INT NOT NULL,
  `user_id` INT NOT NULL,
  `bid_price` DECIMAL(10,2) COMMENT '出价',
  `deposit` DECIMAL(10,2) COMMENT '保证金',
  `bid_time` DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## 4. 非功能需求

### 4.1 性能要求

| 指标 | 目标值 | 测试场景 |
|-----|--------|---------|
| **并发支持** | 500+ 人同时在线 | 开学季/毕业季高峰期 |
| **API 响应时间** | < 500ms (P95) | 商品列表查询、订单创建 |
| **首页加载时间** | < 2s | 小程序冷启动 |
| **数据库查询** | < 100ms | 索引优化（user_id, goods_id, order_id）|
| **缓存命中率** | > 80% | Redis 缓存热门商品、信用等级 |

**技术方案:**
- **缓存策略**: Redis 缓存首页商品列表（TTL 30 分钟）、用户信用等级（TTL 1 小时）
- **数据库优化**: MySQL 索引优化、分页查询（PageHelper 插件）
- **CDN 加速**: 商品图片上传至阿里云 OSS，配置 CDN 加速

### 4.2 安全性

| 安全点 | 实现方案 |
|-------|---------|
| **数据加密** | 学号、真实姓名使用 AES-256 加密存储 |
| **支付安全** | 微信支付官方 SDK，订单金额二次校验防篡改 |
| **接口鉴权** | JWT Token 验证，Token 有效期 30 天 |
| **防刷机制** | 验证码（图形验证码）、频率限制（1 分钟内最多发布 3 个商品）|
| **敏感操作** | 封禁账号、修改权限需二次确认，操作日志记录 IP 和时间 |

### 4.3 兼容性

| 平台 | 要求 |
|-----|------|
| **小程序基础库** | ≥ 2.10.0 |
| **微信版本** | ≥ 7.0.0 |
| **后端 JDK** | OpenJDK 8+ |
| **数据库** | MySQL 8.0 / TiDB Cloud |
| **浏览器（管理后台）** | Chrome ≥ 90, Safari ≥ 14 |

## 7. UX 设计原则

### 7.1 小程序交互原则

1. **轻量化操作**: 每个功能不超过 3 次点击到达
2. **校园场景适配**: 
   - 教材搜索支持课程名（而非 ISBN）
   - 配送选择默认同楼栋快递员
   - 校区切换（成都/雅安/都江堰）
3. **信任可视化**: 
   - 卖家主页显著展示信用等级和评价
   - 商品详情页显示"已成交 X 笔"
4. **社交属性**: 
   - 买卖双方聊天支持位置分享（自提地点）
   - 交易完成后推荐关注对方（建立校园人脉）

## 8. 不包含范围（Out of Scope）

以下功能不在 MVP 阶段实现，可作为后续迭代方向：

- ❌ **多商户入驻**: 当前仅支持学生个人发布，不支持校外商家
- ❌ **跨校交易**: 限定四川农业大学三校区内交易，不支持与其他高校互通
- ❌ **AI 推荐算法**: MVP 使用简单的时间排序和分类筛选，不引入协同过滤
- ❌ **直播带货**: 暂不支持小程序直播功能
- ❌ **分期付款**: 仅支持全额微信支付，不提供分期服务
- ❌ **国际化**: 仅支持简体中文

## 10. 风险评估与应对

| 风险 | 影响 | 概率 | 应对措施 |
|-----|------|------|---------|
| **用户冷启动** | 初期交易量少 | 高 | 开学季地推 + 前 1000 单免配送费 |
| **快递员不足** | 配送服务覆盖率低 | 中 | 与学生会合作招募 + 激励机制（积分翻倍）|
| **学号验证成本** | 人工审核效率低 | 中 | Phase 1 人工审核，Phase 2 对接教务系统 API |
| **微信支付手续费** | 每笔 0.6%，成本较高 | 低 | 初期平台承担，后期考虑拍卖手续费分摊 |
| **违规内容泛滥** | 平台声誉受损 | 中 | AI 图片识别 + 敏感词库 + 用户举报 |
